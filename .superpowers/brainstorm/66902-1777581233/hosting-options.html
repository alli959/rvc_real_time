Cloud Hosting Options for MorphVox

Budget: $80 alert / $100 alert / $150 hard cap — GPU required for all services

A

Vast.ai Persistent Pod — Full Stack

~$60–100/mo — Single RTX 3090 running your entire Docker Compose stack

Pros

  • Cheapest GPU option (community cloud marketplace)
  • Your existing docker-compose.prod.yml works as-is
  • Single deployment target — simple CI/CD
  • Full control over the machine
  • 24GB VRAM for inference + training

Cons

  • Community hardware — host can evict you (rare but possible)
  • Need to handle instance migration if evicted
  • Variable availability depending on demand
  • No built-in redundancy or auto-restart
  • Networking: need Cloudflare Tunnel for HTTPS

Architecture: Vast.ai pod → Cloudflare Tunnel → Users

B

Hybrid: Hetzner VPS + RunPod Serverless GPU

~$30–80/mo — Split non-GPU (always-on VPS) from GPU (pay-per-use)

Pros

  • Cheapest when GPU is idle (scales to $0 GPU cost)
  • Web/API always responsive (no cold starts for pages)
  • Hetzner is reliable European hosting with great networking
  • Built-in cost scaling — pay only for actual GPU time
  • Budget alerts easy via RunPod spending limits

Cons

  • Requires refactoring voice-engine into serverless endpoints
  • 30–60s cold start on first GPU request
  • Two providers to manage
  • More complex deployment (VPS + serverless)
  • WebSocket real-time streaming harder in serverless

Architecture: Hetzner (web/API/DB) → RunPod Serverless (GPU inference)

C

RunPod Persistent Pod — Full Stack

~$100–150/mo — Managed GPU pod with better reliability than Vast.ai

Pros

  • More reliable than Vast.ai (managed infrastructure)
  • Persistent network storage survives restarts
  • Docker-compose compatible (run full stack)
  • Built-in spending limits and cost alerts
  • Better uptime guarantees — pods auto-restart

Cons

  • More expensive (~$0.40–0.50/hr for 3090 community)
  • At budget ceiling ($100–150/mo)
  • Still need Cloudflare Tunnel for custom domain HTTPS
  • Storage costs extra on top of compute
  • Less cost-efficient if GPU often idle

Architecture: RunPod pod → Cloudflare Tunnel → Users
